# ほしさん専用やらかしリスト

**起動時に必ず読むこと。ここに書いてあるミスを繰り返したら方針違反です。**

## 過去のミスと対策

### 【最重要：翔伍さんがOKを出したLP・デザインを絶対に壊すな】
- 翔伍さんが認めたLPのデザイン・レイアウト・アニメーションを、後から勝手に変更して壊してしまった
- **これは最も重大な方針違反。良いものを壊すのは、何もしないより悪い**
- **【実例・2026-04-06】リースバックLPを修正中にバックアップを取らずに確定版を破壊。画像パスも壊れて表示不能になった。翔伍さんが手動で復元する事態に。二度と繰り返すな。**
- **対策：**
  - 翔伍さんからOKが出たLPは「確定版」として記録すること
  - 確定版のHTML/CSSは必ずバックアップを取ってから修正に入ること
  - 確定版のデザイン（配置・色・フォント・アニメーション）は変更禁止
  - 修正は指示された箇所だけ。他の箇所は一切触らない
  - 修正する前に「この箇所だけ変えます。他は触りません」と翔伍さんに宣言すること
- **記録方法：progress.mdに以下のように記録せよ**
  ```
  ## 確定版（翔伍さんOK済み・変更禁止）
  - ファイル名: ○○.html
  - バックアップ: ○○_backup_YYYYMMDD.html
  - OKもらった日付: YYYY-MM-DD
  - 確定箇所: （デザイン全体 or 特定セクション）
  ```
- **修正作業の手順：**
  1. 確定版のバックアップを取る
  2. 修正箇所だけをリストアップして翔伍さんに見せる
  3. OKが出た箇所だけ修正する
  4. 修正後、確定版と見比べて壊れていないことを確認する
  5. Before→Afterで報告する

### 【LPの配置修正が反映されない】
- 翔伍さんから配置の修正指示を受けても、なかなか反映されない
- **対策：修正指示を受けたら、指示内容を1つずつ箇条書きにして、全項目をチェックリスト化すること**
- **提出前チェック：修正前と修正後のスクショを並べて、各指示項目が反映されていることを目視確認してから提出すること**
- **「直しました」だけの報告は禁止。「指示①〇〇→修正済み（スクショ）、指示②△△→修正済み（スクショ）」と1つずつ対応を示すこと**

## 提出前チェックリスト（毎回確認）
- [ ] 確定版のバックアップを取ったか ←【最優先】
- [ ] 確定版のデザインを壊していないか ←【最優先】
- [ ] 修正は指示された箇所だけか（他を触っていないか）
- [ ] Before→Afterのスクショを用意したか
- [ ] スマホ表示でも崩れていないか

**このチェックリストを全部クリアしてから翔伍さんに提出すること。**

---

## 翔伍さんからの修正指示が来たら

1. 指示内容をこのファイルに追記する
2. 確定版のバックアップを取る
3. 指示を1項目ずつチェックリスト化する
4. 1項目ずつ修正し、チェックを入れる
5. 確定版と見比べて壊れていないことを確認する
6. 全項目チェック済みになってから提出する
7. Before→Afterのスクショを添えて報告すること
